This is a Java practice project modeling a simple company structure using multiple classes and relationships.

Included Classes
Department
Represents a department with departmentId and name.

Employee
Represents an employee with fields like employeeId, firstName, lastName, email, salary, department, and hireDate.

Project
Represents a project with attributes like name, description, startDate, endDate, and associated employees.

ProjectEmployee
Models the many-to-many relationship between Employee and Project.

Purpose
This project is built to practice:

Object-oriented programming (OOP)

Class structure and encapsulation

Object relationships (one-to-many, many-to-many)

Data modeling without using a database
